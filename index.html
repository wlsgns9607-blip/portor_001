B2C 세차용품 전문 커머스 플랫폼

세차 매니아들을 위한 직관적이고 빠른 쇼핑 경험을 제공합니다. 프로페셔널한 디테일링 제품을 한 곳에서 만나보세요.

프로젝트 개요

🎯 프로젝트 기획 배경

2년간의 디테일링 세차 실무 경험을 바탕으로, 제가 가장 잘 알고 애정하는 분야인 '세차 용품 전문 쇼핑몰'을 타겟으로 기획했습니다. 단순한 화면 구현을 넘어, 토스 페이먼츠 결제 API를 연동해 실무에 가까운 커머스 플로우를 구축하는 데 중점을 두었습니다.

👤 역할 및 AI 협업

UX/UI 디자인 - 피그마로 와이어프레임 및 고피델리티 디자인
AI 협업 (Claude) - AI와 협력하여 HTML, CSS, JavaScript 프론트엔드 UI 구현

💻 기술 스택

기술 스택:

HTML CSS JavaScript Toss API OAuth 2.0 Figma

핵심 기능

상품 상세 페이지

세차용품의 특성을 한눈에 파악할 수 있도록 직관적인 UI/UX를 설계했습니다. 사용자가 제품 정보를 쉽게 확인하고 빠르게 장바구니에 담을 수 있습니다.

토스 페이먼츠 결제 연동

Toss Payments API를 연동하여 장바구니부터 실제 테스트 결제까지 완벽한 이커머스 결제 플로우를 구현했습니다.

관리자(Admin) 대시보드

일반 고객 환경과 완벽히 분리된 운영자 전용 페이지를 구축하여, 상품 등록 및 매출 현황 등 주요 데이터를 한눈에 관리할 수 있도록 설계했습니다.

멀티 소셜 로그인 (OAuth 2.0)

구글, 네이버, 카카오 3대 소셜 로그인 API를 연동했습니다. 번거로운 회원가입 절차 없이 클릭 한 번으로 가입과 로그인을 지원하여 사용자 접근성을 크게 향상시켰습니다.

직관적인 장바구니 (사이드바 UI)

상품 상세 페이지에서 제품을 담으면 우측에서 스르륵 열리는 사이드바 형태의 장바구니를 구현했습니다. 선택한 제품들의 수량과 총 결제 금액을 한눈에 확인하고 '결제하기'로 바로 이어지도록 하여 이탈률을 최소화했습니다.

반응형 웹 디자인

Bubble Mate는 모든 기기에서 완벽하게 작동하도록 설계되었습니다. 아래 탭을 통해 다양한 화면 크기에서의 레이아웃을 확인하세요.

트러블슈팅 (문제 해결 과정)

프로젝트를 기획하고 개발하면서 마주친 핵심 문제들과 이를 해결한 과정입니다.

🚨 문제 상황:

PC 화면을 기준으로 기획하고 AI(클로드)와 협업하여 코드를 짰는데, 막상 모바일에서 확인해보니 이미지와 글씨가 좌우로 찌그러져서 가독성이 크게 떨어졌습니다.


💡 해결 과정:

HTML 내부에 하드코딩되어 있던 인라인 스타일(inline-style)이 원인임을 파악했습니다. 인라인 스타일을 모두 CSS 파일로 분리하고, @media (max-width: 768px) 반응형 쿼리를 활용해 모바일에서는 좌우 배치가 아닌 위아래(1줄)로 자연스럽게 떨어지도록 CSS Grid를 수정하여 사용자 가독성을 높였습니다.

🚨 문제 상황:

AI에게 전체 코드를 주고 특정 부분 수정을 요청할 때마다, 멀쩡하던 다른 섹션의 디자인이나 애니메이션이 풀려버리는 문제가 잦았습니다.


💡 해결 과정:

프롬프트 엔지니어링 방식을 개선했습니다. 한 번에 너무 많은 것을 요구하지 않고, 컴포넌트 단위(예: "네비게이션 바 영역만 수정해줘")로 쪼개서 질문했습니다. 또한 기존 코드를 해치지 말라는 명확한 제약 조건(Instruction)을 주어 AI와의 협업 효율을 극대화했습니다.

🚨 기획 고민:

초기 기획에서는 '장바구니 담기'를 누르면 장바구니 페이지로 화면이 아예 넘어가버렸습니다. 사용자가 쇼핑을 멈추게 되어 이탈률이 걱정되었습니다.


💡 해결 과정:

페이지 이동 없이 우측에서 부드럽게 열리는 '사이드바(Sidebar) 장바구니' 형태로 UX를 재설계했습니다. 사용자는 현재 보던 상품 페이지에 머무르면서 담긴 내역과 총액만 빠르게 확인하고 쇼핑을 이어갈 수 있도록 개선했습니다.
